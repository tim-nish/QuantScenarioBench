- (direction by user) User priority: architecture should focus on a clean, extensible design rather than implementation details. The State-Space Interface (observation/latent_state/metadata/optional randomness) and its extension points (how new Market Models plug in without touching simulate() or the export pipeline) are the highest-priority architectural concerns
- (version) Verified (web, 2026-06-30): diffrax 0.7.2 on PyPI, requires Python >=3.11, jax>=0.4.38, equinox>=0.11.10. diffrax solvers/terms subclass equinox.Module; eqx.Module is simultaneously an ABC (abstractmethod works) and a registered JAX pytree (jit/vmap-compatible natively); eqx.field(static=True) marks non-array fields as pytree aux_data rather than traced leaves. diffrax.VirtualBrownianTree generates Brownian increments on-the-fly from a tree-search algorithm (no full noise-path materialization), passed as the control to ControlTerm; diffrax.UnsafeBrownianPath is a faster but determinism/backprop-restricted alternative
- (question) If diffrax is adopted as the solver foundation, PRD Cross-Cutting NFR [ASSUMPTION: Python 3.10+] is inaccurate - diffrax 0.7.2 requires Python >=3.11. Needs reconciliation back to PRD if diffrax is chosen
- (decision by user) AD: State-Space Interface is an equinox.Module ABC (abstract methods enforced at construction; natively a JAX pytree via eqx, no separate pytree-registration step). Binds: every Market Model. Prevents: structural-typing-only contracts that skip eqx's pytree/jit integration
- (decision by user) AD: Scenario is an equinox.Module pytree dataclass. observation and latent_state are dynamic (traced) fields; metadata is a static field (eqx.field(static=True), pytree aux_data, not traced). Binds: simulate()'s return type and the export pipeline's input type. Prevents: metadata (strings/ints) leaking into traced leaves and breaking jit
- (decision by user) AD: Randomness defaults to diffrax.VirtualBrownianTree (no full noise-path materialized). Materialized randomness (FR-5) is an explicit, separate opt-in construction path, not a flag toggled inside one shared code path. Binds: the Solver Layer's randomness construction. Prevents: per-model divergence in how Brownian motion is constructed
- (decision by user) AD: An internal Solver Layer wraps diffrax; diffrax is never called directly from Market Model code. The Solver Layer is the only place that constructs diffrax Terms/SaveAt/solver choice/Brownian motion. Binds: all Market Model implementations (FR-7,8,9) and the State-Space Extensibility Contract (FR-10). Prevents: two Market Models choosing incompatible TimeGrid-to-SaveAt mapping, noise construction, or precision conventions
- (decision by user) AD: Dataset export is fully generic over the Scenario schema (pytree-flattening derives Parquet columns from observation/latent_state/metadata); the export pipeline never imports or branches on a specific Market Model. Binds: Feature 4.4 (FR-12 through FR-15). Prevents: per-model export hooks reintroducing model-specific coupling into the pipeline
- (decision by user) AD: equinox is adopted as a project-wide convention (not just for diffrax integration) - all core pytree-bearing types (Market Model configs, Scenario) are eqx.Module. Binds: PRD's 'JAX PyTrees' requirement for Market Model parameter dataclasses (FR realized via eqx.Module rather than hand-rolled tree_util registration)
- (decision by user) Correction to PRD Cross-Cutting NFR: Python runtime target is Python >=3.11 (not >=3.10 as PRD's [ASSUMPTION] stated), driven by diffrax 0.7.2's minimum requirement. To be reconciled back into prd.md
- (decision by user) AD: float64 (JAX x64 mode enabled project-wide) is the fixed precision policy for v1, set once at the Solver Layer/process level. Binds: all Market Model simulation and FR-7/8 correctness-tolerance checks. Prevents: per-model precision divergence making cross-model comparison or tolerance checks meaningless
- (decision by user) AD: Metadata's minimum guaranteed field set is: seed, PRNG key info, Market Model name + version-stamped parameter values, TimeGrid reference, library version, dataset version (FR-14), generation timestamp. Binds: Scenario.metadata (static eqx field) and dataset card generation (FR-15). Resolves PRD Open Question 2
